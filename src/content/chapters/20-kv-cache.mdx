# Autoregressive Decode and the KV-Cache

**Where we are.** Parts 3–4 trained R1 at scale. Part 5 serves it — and serving obeys a completely
different economics from training, because it produces *one token at a time*. This chapter introduces the
data structure that makes that tractable, the KV-cache, and shows why it, not the weights, dominates the
memory a decode step moves.

## 1. Generation without a cache is quadratic

<Invariant note="the redundant recomputation">
An autoregressive decoder generates token $t$ by attending over tokens $1..t{-}1$. Computed naïvely, each
new token recomputes the keys and values for *all* previous tokens — so generating a sequence of length
$T$ costs $O(T^2)$ attention work, most of it re-deriving keys and values that have not changed since the
step that first produced them. The past is fixed; recomputing it is pure waste.
</Invariant>

## 2. The cache makes each token linear

<Invariant note="the KV-cache">
Because a past token's key and value never change, they can be computed once and stored. The **KV-cache**
holds, per layer, the keys and values of every token generated so far; a new token computes only *its own*
query, key, and value, appends the latter two to the cache, and attends against the whole cache. Per-token
work drops from $O(t)$ recomputation to $O(1)$ new projection plus an $O(t)$ read of the cache — turning
quadratic generation into linear. The cost moves from *compute* to *memory*: the cache must be stored and
streamed every step.
</Invariant>

This is the trade that defines inference: we spend memory to erase redundant compute, and in doing so make
the decode step *memory-bound* — exactly the regime the roofline warned streams bytes, not FLOPs.

## 3. The cache is the inference memory budget

<Invariant note="KV-cache size and why GQA matters here">
The cache holds $2$ (K and V) $\times\ n_\text{layers} \times n_\text{kv} \times d_\text{head}$ values per
token per sequence, times the bytes per value. It grows linearly with sequence length and with the number
of concurrent sequences, and at long context or high concurrency it dwarfs the model weights. This is why
the architecture chapter's **grouped-query attention** was an inference decision: using $n_\text{kv}=2$
instead of $8$ heads shrinks the cache — and thus the bytes each decode step streams — by $4\times$.
</Invariant>

The KV-cache is where R1-tiny's Part-2 GQA choice pays off: the $4\times$ smaller cache is a $4\times$
reduction in the dominant decode-time memory traffic.

<Renewable last_measured="2026-07-17" measured_on="Tier-0 RTX 2070 SUPER" source="https://github.com/brandon-behring/transformers-systems-lab">
The lab's serving path caches K/V per layer and generates incrementally; its acceptance test pins the
invariant that **cached incremental decode produces bit-identical greedy tokens to a full recompute** — the
cache is a pure efficiency optimization, changing what is stored, never what is produced.
</Renewable>

**Where this goes.** The cache splits inference into two phases with opposite characters — filling the
cache for a prompt, and extending it one token at a time. The next chapter shows they sit on opposite sides
of the roofline.
